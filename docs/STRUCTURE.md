Інтеграція з картографічними сервісами (Reverse Geocoding)Для покращення користувацького досвіду (UX) було впроваджено функцію автоматичного визначення адреси. При взаємодії з картою (бібліотека Leaflet) система виконує асинхронний запит до API Nominatim (OpenStreetMap).Результат: Користувачу не потрібно вводити адресу вручну; система автоматично отримує назву вулиці та номер будинку за географічними координатами ($lat$, $lng$).2. Архітектура взаємодії компонентівРеалізовано передачу даних між незалежними компонентами Angular за допомогою декораторів @Input() та @Output(). Ланцюжок передачі даних виглядає наступним чином:MapComponent: отримує координати та адресу.RequestFormComponent: приймає ці дані та валідує їх через ReactiveForms.VolunteerRequestService: формує GraphQL-мутацію та відправляє її на сервер.3. Система детального перегляду (Side Panel/Drawer)Замість стандартних спливаючих вікон (popups) було розроблено інтерактивну бічну панель (Sidebar). Це рішення дозволило:Використовувати більше простору для детального опису волонтерської потреби.Відображати статус запиту та категорію в зручному для читання форматі.Реалізувати плавні анімації появи панелі через CSS Transitions.4.
